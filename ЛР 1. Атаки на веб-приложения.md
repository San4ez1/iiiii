# **2.1.**  **Нарушение контроля доступа**

**A01:2021 – Broken Access Control**
-        это тип атаки, при котором злоумышленники обходят систему контроля доступа и получают несанкционированный доступ к защищенным ресурсам или данным. Неработающий контроль означает, что механизмы контроля доступа, предполагаемые для защиты информации, не функционируют должным образом, что делает систему уязвимой для атак.
Атака "restrict folder access" (ограничение доступа к папкам) заключается в том, что злоумышленники пытаются получить несанкционированный доступ определенным папкам на компьютере или сервере, ограниченным с точки зрения доступа. Они могут использовать различные методы чтобы обойти механизмы контроля доступа и получить доступ к защищенным данным.

К примеру,
1.    Прямой доступ к защищенным ресурсам: Злоумышленник может попытаться получить доступ к защищенным данным или функциональности, обходя проверку прав доступа. - **CWE-285**: Improper Authorization, **CWE-284**: Неправильный контроль доступа
2.    Изменение параметров запроса: Злоумышленник может изменить параметры запроса (например, идентификатор пользователя в URL), чтобы получить доступ к данным другого пользователя или выполнить действия от его имени.
3.    Использование недействительных сеансов: Злоумышленник может использовать украденные или поддельные сеансовые идентификаторы для получения доступа к учетным записям других пользователей.

# **2.12.** **Удаленное выполнение команд**

Уязвимости удаленного выполнения команд (Remote Code Execution vulnerabilities) могут соответствовать нескольким общепринятым классификациям CWE (Common Weakness Enumeration). Некоторые из наиболее подходящих CWE для уязвимостей удаленного выполнения команд включают:

# **2.14.** **Внешние сущности XML (XXE)**

CWE-611: Improper Restriction of XML External Entity Reference - Неправильное ограничение ссылки на внешний объект XML

-       Эта уязвимость возникает, когда XML-парсер позволяет ссылаться на внешние сущности или файлы, что может привести к различным видам атак, включая обнаружение файловой системы, денежное отчуждение, атаки на конфиденциальность данных и другие.

-       Суть уязвимости CWE-611 заключается в том, что приложение не должно разрешать внешние ссылки на сущности в XML без должного контроля. Если злоумышленник может внедрить вредоносный XML-код с ссылками на внешние сущности, это может привести к серьезным последствиям, таким как чтение конфиденциальных данных, выполнение нежелательных операций и даже удаленное выполнение кода.

-       Для предотвращения уязвимости CWE-611 необходимо использовать безопасные методы обработки XML, такие как отключение поддержки внешних сущностей (защита от XXE), фильтрация и проверка входных данных, а также использование безопасных API для работы с XML.
